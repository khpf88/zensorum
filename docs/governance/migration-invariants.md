# Migration Invariants

The following invariants MUST hold across the TypeScript (Active) to Go (Target) migration to ensure system integrity:

1. **Deterministic Event Ordering:** The event log replay must produce the exact same sequence of state changes.
2. **Replay Equivalence:** Replaying the same event log on both runtimes must result in identical final states.
3. **FRCS Validation Consistency:** All governance rules must be evaluated identically across both runtimes.
4. **State Reconstruction Parity:** State snapshots generated by both runtimes must be interchangeable.
5. **Idempotency Guarantees:** Idempotency keys must be respected identically to prevent duplicated event processing during replay or ingestion.
